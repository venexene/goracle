# Типы данных в Go

Все данные, которые хранятся в памяти, по сути представляют просто набор битов. Именно **тип данных** определяет, как эти биты будут интерпретироваться и какие операции с ними допустимы. Язык Go является **статически типизированным** — тип каждой переменной известен на этапе компиляции и не может измениться во время исполнения.

Статическая типизация даёт три ключевых преимущества:

* **Безопасность** — компилятор отлавливает несовместимые операции с типами до запуска программы, а не во время исполнения.
* **Производительность** — зная точный размер и лэйаут данных, компилятор генерирует оптимальный машинный код без накладных расходов на динамическую диспетчеризацию.
* **Читаемость** — сигнатуры функций и объявления переменных явно документируют, с какими данными работает код.

> **Зачем это Go-разработчику.** Статическая типизация — одно из основных проектных решений языка. Она позволяет в крупных проектах уверенно рефакторить код: компилятор найдёт все места, где типы перестали совпадать.

Go имеет богатый набор встроенных типов, а также позволяет определять собственные. Ниже рассмотрены все встроенные типы — от простых числовых к строкам, а затем механизмы приведения и создания пользовательских типов.

***

## 1. Числовые типы

Go предоставляет типы для целых чисел, чисел с плавающей точкой и комплексных чисел. Каждый числовой тип имеет фиксированный размер в памяти, известный на этапе компиляции.

### Целочисленные типы

Целочисленные типы делятся на **знаковые** (могут хранить отрицательные числа) и **беззнаковые** (только неотрицательные). Разрядность `N` определяет размер в битах, а диапазон значений вычисляется по формулам:

* Знаковые: $[-2^{N-1},\ 2^{N-1}-1]$
* Беззнаковые: $[0,\ 2^{N}-1]$

Сводка всех целочисленных типов:

| Тип      | Размер       | Диапазон                                               |
| -------- | ------------ | ------------------------------------------------------ |
| `int8`   | 1 байт       | –128 … 127                                             |
| `int16`  | 2 байта      | –32 768 … 32 767                                       |
| `int32`  | 4 байта      | –2 147 483 648 … 2 147 483 647                         |
| `int64`  | 8 байт       | –9 223 372 036 854 775 808 … 9 223 372 036 854 775 807 |
| `uint8`  | 1 байт       | 0 … 255                                                |
| `uint16` | 2 байта      | 0 … 65 535                                             |
| `uint32` | 4 байта      | 0 … 4 294 967 295                                      |
| `uint64` | 8 байт       | 0 … 18 446 744 073 709 551 615                         |
| `byte`   | 1 байт       | 0 … 255 (синоним `uint8`)                              |
| `rune`   | 4 байта      | –2 147 483 648 … 2 147 483 647 (синоним `int32`)       |
| `int`    | 4 или 8 байт | Зависит от платформы (32- или 64-разрядная)            |
| `uint`   | 4 или 8 байт | Зависит от платформы                                   |

#### `byte` и `rune`

**`byte`** — синоним `uint8`. Используется, когда нужно подчеркнуть, что значение — это именно байт (например, элемент байтового среза при работе с бинарными данными).

**`rune`** — синоним `int32`, но семантически представляет **Unicode-кодовую точку** (code point). Одна руна может кодировать любой символ Unicode. Именно руны возвращает `range` при итерации по строке:

```go
for i, r := range "Привет" {
    // r имеет тип rune и содержит один символ Unicode
}

```

#### `int` и `uint`

Типы `int` и `uint` имеют **наиболее эффективный размер** для текущей платформы — 32 бита на 32-разрядных системах и 64 бита на 64-разрядных. Это основной тип для представления целых чисел в программе. Различные компиляторы могут предоставлять разный размер для этих типов даже на одной платформе, но в рамках одной сборки размер фиксирован.

> **Зачем это Go-разработчику.** Числовые типы в Go: `int`/`uint` по умолчанию для счётчиков и индексов; типы фиксированной разрядности — для бинарных протоколов и сериализации. `float64` для большинства вычислений с плавающей точкой; НЕ используйте float для денег — храните копейки в `int`. Комплексные числа — для научных вычислений, в бизнес-коде не нужны.

### Числа с плавающей точкой

Для представления дробных чисел Go предоставляет два типа, соответствующие стандарту IEEE 754:

| Тип       | Размер  | Диапазон (положительные)       | Точность             |
| --------- | ------- | ------------------------------ | -------------------- |
| `float32` | 4 байта | от \~1.4×10⁻⁴⁵ до  \~3.4×10³⁸  | \~6 десятичных цифр  |
| `float64` | 8 байт  | от \~4.9×10⁻³²⁴ до \~1.8×10³⁰⁸ | \~15 десятичных цифр |

Оба типа поддерживают специальные значения IEEE 754:

* **`NaN`** (Not a Number) — результат некорректной операции (например, `0.0 / 0.0`). Любое сравнение с `NaN` всегда ложно, включая `NaN == NaN`.
* **`±Inf`** — положительная и отрицательная бесконечность (результат деления на ноль, переполнения).

### Комплексные числа

Go — один из немногих mainstream-языков со встроенной поддержкой комплексных чисел:

| Тип          | Вещественная и мнимая части | Размер  |
| ------------ | --------------------------- | ------- |
| `complex64`  | `float32`                   | 8 байт  |
| `complex128` | `float64`                   | 16 байт |

Встроенные функции для работы с комплексными числами:

* `complex(r, i)` — создаёт комплексное число из вещественной и мнимой частей.
* `real(c)` — возвращает вещественную часть.
* `imag(c)` — возвращает мнимую часть.

Арифметические операции (`+`, `-`, `*`, `/`) работают с комплексными числами напрямую.

## 2. Логический тип

Логический тип **`bool`** принимает одно из двух значений: `true` (истина) или `false` (ложь). Он используется:

* Как результат операторов сравнения (`==`, `!=`, `<`, `>`, `<=`, `>=`) и логических операторов (`&&`, `||`, `!`).
* Как условие в конструкциях `if`, `for`.
* Как флаг состояния.

В отличие от C, в Go **нет неявного преобразования** между `bool` и целыми числами. Нельзя написать `if 1 { ... }` или `if x { ... }` для целого `x` — условие обязано быть выражением типа `bool`.

> **Зачем это Go-разработчику.** Явная булева логика исключает целый класс ошибок, связанных с путаницей `0`/`1`/`true`/`false`. Именованные булевы переменные (`isReady`, `hasPermission`) делают код самодокументируемым — читатель сразу понимает смысл условия.

## 3. Строки

Строки в Go представлены типом **`string`**. Строковый литерал — последовательность символов, заключённая в двойные кавычки:

```go
s := "Hello, мир"

```

### Строки и UTF-8

Строка в Go — это **неизменяемая последовательность байт**. Каждый байт представляет часть символа в кодировке **UTF-8** — кодировке переменной длины, способной представить любой символ Unicode. Следствия:

* Один символ (руна) может занимать от 1 до 4 байт.
* Функция `len(s)` возвращает количество **байт**, а не символов.
* Для итерации по символам используйте `for ... range` — она декодирует руны:

```go
s := "Привет"
fmt.Println(len(s))           // 12 (байт)
for _, r := range s {
    fmt.Println(string(r))    // П, р, и, в, е, т (руны)
}

```

### Неизменяемость

После создания строки её содержимое **нельзя изменить**. Операция `s[0] = 'X'` невозможна. При «изменении» строки всегда создаётся новая:

```go
s := "hello"
s = "Hello" // новая строка, старая будет собрана сборщиком мусора

```

Для эффективной конкатенации множества строк используйте `strings.Builder`, а не повторяющийся `+`.

### Управляющие последовательности

Строки в двойных кавычках поддерживают escape-последовательности:

| Последовательность | Значение        |
| ------------------ | --------------- |
| `\n`               | Перевод строки  |
| `\r`               | Возврат каретки |
| `\t`               | Табуляция       |
| `\"`               | Двойная кавычка |
| `\\`               | Обратный слеш   |

### Raw strings (сырые строки)

Литерал в обратных кавычках (backticks) задаёт **сырую строку** — escape-последовательности в ней не обрабатываются, а переносы строк сохраняются как есть:

```go
raw := `строка с \n без обработки
и с реальным переносом строки`

```

Это удобно для регулярных выражений, JSON-схем, SQL-запросов и многострочного текста.

> **Зачем это Go-разработчику.** Понимание разницы между байтами и рунами критически важно при обработке текста от пользователей. `len()` может обмануть — всегда проверяйте, что именно вы считаете. `strings.Builder` вместо наивной конкатенации в цикле даёт прирост производительности на порядки. Raw strings избавляют от лавины обратных слешей в регулярках.

## 4. Значения по умолчанию (zero values)

Если переменная объявлена, но ей не присвоено значение явно, она автоматически получает **zero value** — значение по умолчанию для своего типа. Это не «мусор» из памяти, а детерминированное, гарантированное компилятором значение:

| Тип                                                                       | Zero value                          |
| ------------------------------------------------------------------------- | ----------------------------------- |
| Все числовые типы (`int`, `float64`, `complex128`, `byte`, `rune` и т.д.) | `0`                                 |
| `bool`                                                                    | `false`                             |
| `string`                                                                  | `""` (пустая строка)                |
| Указатели, срезы, карты, каналы, интерфейсы, функции                      | `nil`                               |
| Структуры                                                                 | Все поля имеют свои zero values     |
| Массивы                                                                   | Все элементы имеют свои zero values |

Идиома Go: **«сделай zero value полезным»**. Например, `sync.Mutex` можно использовать сразу после объявления без явной инициализации, а пустой срез `var s []int` равен `nil`, но при этом `len(s) == 0` и по нему можно итерироваться.

> **Зачем это Go-разработчику.** Zero values избавляют от неинициализированных переменных — в Go нет проблемы «undefined behavior» из C/C++. Вы можете объявить переменную и сразу начать её использовать в безопасном начальном состоянии. При проектировании своих типов стремитесь к тому, чтобы zero value был осмысленным и готовым к работе.

## 5. Приведение типов

Go **никогда не преобразует типы неявно** — любое преобразование должно быть явным. Синтаксис: `T(x)`, где `T` — целевой тип, `x` — значение.

### Числовые преобразования

```go
var i int32 = 42
var f float64 = float64(i) // явное приведение int32 → float64
var u uint8 = uint8(i)     // явное приведение int32 → uint8

```

При приведении из более широкого типа в более узкий происходит **усечение** (truncation) — старшие биты отбрасываются, что может привести к потере данных:

```go
var big int16 = 300
var small uint8 = uint8(big) // 300 не помещается в uint8 → результат 44

```

Приведение `float64` → `int` отбрасывает дробную часть (не округляет):

```go
var f float64 = 3.9
var i int = int(f) // i == 3

```

### Приведение строк и байтов

Для преобразования строки в срез байт (и обратно) используется тот же синтаксис `T(x)`:

```go
s := "hello"
b := []byte(s)   // строка → байтовый срез
s2 := string(b)  // байтовый срез → строка

```

Это **не каст**, а полноценное копирование данных — строка и срез байт не разделяют память.

### Вывод типов

При использовании короткого объявления `:=` тип выводится автоматически на основе значения справа:

```go
i := 42          // int
f := 3.14        // float64
s := "hello"     // string
b := true        // bool
r := '€'         // rune

```

> **Зачем это Go-разработчику.** Явное приведение — это осознанное проектное решение. Оно заставляет разработчика подтвердить: «да, я знаю, что преобразую `int` в `uint8`, я осознаю риск переполнения». Это устраняет скрытые баги, которые в языках с неявным приведением проявляются только в production. Если вам нужны безопасные преобразования с проверкой переполнения — смотрите пакет `math/bits` (Go 1.18+) или функции вида `strconv.Atoi` для строк.

## 6. Псевдонимы типов и пользовательские типы

Go позволяет определять собственные типы на основе существующих. Это не просто синоним — это **новый, отдельный тип** со своей областью видимости методов.

### Определение нового типа

```go
type UserID int64
type Celsius float64

```

После такого определения `UserID` и `int64` — разные типы. Присвоить `int64` переменной типа `UserID` без явного приведения нельзя:

```go
var uid UserID
var i int64 = 42
uid = UserID(i) // OK: явное приведение
// uid = i      // Ошибка компиляции: cannot use i (type int64) as type UserID

```

Это обеспечивает **типобезопасность на уровне предметной области** — компилятор не даст перепутать идентификатор пользователя с количеством товаров на складе.

### Псевдоним типа (type alias)

Начиная с Go 1.9, можно объявить **псевдоним** — альтернативное имя для существующего типа. Синтаксис: `type Name = ExistingType` (со знаком `=`):

```go
type MyInt = int // MyInt — это псевдоним, а не новый тип

```

`MyInt` и `int` полностью взаимозаменяемы — приведение не требуется. Псевдонимы используются в основном для крупномасштабного рефакторинга, когда тип перемещается между пакетами, но нужно сохранить обратную совместимость.

### Методы на пользовательских типах

На определённый тип можно объявлять методы:

```go
type Celsius float64

func (c Celsius) ToFahrenheit() float64 {
    return float64(c)*9/5 + 32
}

temp := Celsius(100)
fmt.Println(temp.ToFahrenheit()) // 212

```

Это даёт Go-разработчику возможность добавлять поведение к доменным типам, не прибегая к наследованию.

> **Зачем это Go-разработчику.** Пользовательские типы — один из главных инструментов обеспечения корректности кода в предметной области. Если в вашей системе есть понятия «метры», «килограммы», «идентификатор заказа» — сделайте для каждого отдельный тип. Компилятор станет вашим союзником: он не даст передать метры в функцию, ожидающую килограммы. Это делает код самодокументированным и предотвращает целый класс логических ошибок.
