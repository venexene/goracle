## Рекомендуемый порядок

1. **Принципы программирования** — усвоить базовые правила дизайна
2. **ООП в Go** — понять, как объектно-ориентированные концепции работают без классов
3. **Чистая архитектура** — научиться строить системы с разделением ответственности
4. **Микросервисная архитектура** — масштабировать подход на распределённые системы
5. **HTTP** — понять основной протокол веба и микросервисов, его реализацию в `net/http`
6. **gRPC** — освоить протокол межсервисного взаимодействия на основе HTTP/2
