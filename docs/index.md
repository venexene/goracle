# Goracle

**Открытая база знаний по внутреннему устройству Go** — от планировщика и сборщика мусора до архитектурных паттернов и структур данных.

Каждая тема разбирается от фундаментальных концепций к практическим выводам: что, как, и главное — **зачем это Go-разработчику**.

## Как читать

Материал построен **от фундамента к деталям**. Каждый документ:

1. Начинается с **базовых концепций** — даже сложные темы вроде сборщика мусора объясняются без предположения, что вы это уже знаете
2. Содержит блоки **«Зачем это Go-разработчику»** — практическая привязка к реальной разработке
3. Использует **диаграммы и живой Go-код** с подсветкой синтаксиса
4. Завершается **практическими выводами** — что делать и чего избегать

Стартовая точка зависит от вашего уровня:

- **Начинающий** → Компьютерные системы → Алгоритмы → ООП в Go → Принципы программирования
- **Middle** → Планировщик Go → Память в Go → Escape Analysis → Сборщик мусора
- **Архитектор** → Чистая архитектура → Микросервисная архитектура

## О проекте

Goracle создаётся одним разработчиком как личная база знаний, открытая для сообщества. Каждый документ — результат глубокой проработки темы, а не беглого конспекта.

Проект активен и пополняется. Следите за обновлениями в [репозитории на GitHub](https://github.com/venexene/goracle).
